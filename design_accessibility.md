# Antenna App — Design: HTML Accessibility and ARIA Improvements

## Goal

Bring generated HTML into conformance with WCAG 2.1 Level A (and selected Level AA) for the three
page types `WriteHTML` (aggregate collection), `WriteHtmlPage` (posts and standalone pages), and
the `WriteItem` helper that builds individual feed article cards.

Decisions: DEC-016 through DEC-021 in `decisions.md`.

---

## Changes by component

### 1. `Generator` struct — new `Lang` field

```go
type Generator struct {
    // ... existing fields ...
    Lang string  // html lang attribute, default "en-US"
}
```

`NewGenerator` sets `Lang = "en-US"`. `LoadConfig` reads an optional `lang:` key from the
generator YAML. Both `WriteHTML` and `WriteHtmlPage` write `<html lang="%s">` with `gen.Lang`.

---

### 2. Skip navigation link — both page types

Inserted as the first element of `<body>`, before any `<header>` or `<nav>`:

```html
<a href="#main-content" class="skip-link">Skip to main content</a>
```

The target `<main id="main-content">` already exists in both `WriteHTML` and `WriteHtmlPage`.

CSS rule required in the default Antenna App stylesheet:

```css
.skip-link {
    position: absolute;
    top: -999px;
    left: -999px;
}
.skip-link:focus {
    position: static;
    top: auto;
    left: auto;
}
```

This hides the link off-screen for mouse users and makes it visible on keyboard focus.

---

### 3. Feed item article structure — `WriteItem`

**Before:**

```html
<article data-published="DATE" data-link="URL" data-pagefind-filter="…">
  <h2>Post Title

(date: 2020-04-11)</h2>
  <p>Description text.</p>
  <address>
    <a href="URL">URL</a>
  </address>
</article>
```

**After:**

```html
<article data-published="DATE" data-link="URL" data-pagefind-filter="…">
  <h2>Post Title</h2>
  <p><time datetime="2020-04-11">2020-04-11</time></p>
  <p>Description text.</p>
  <footer>
    <a href="URL">URL</a>
  </footer>
</article>
```

Changes:
- `(date: DATE)` removed from heading string; heading carries only the title text.
- Date emitted as `<p><time datetime="YYYY-MM-DD">YYYY-MM-DD</time></p>` below the heading.
  If `updated` differs from `pubDate`, both are shown:
  `<time datetime="PUB">PUB</time>, updated: <time datetime="UPD">UPD</time>`.
- `<address>` replaced with `<footer>`.

When `title` is empty the heading still exists with a label-only heading (e.g., `@channel`),
and the `<time>` element is still emitted.

---

### 4. `rel="altenate"` typo fix — `writeHeadElement`

In `writeHeadElement`, the Markdown source alternate link changes:

```go
// Before
"rel":   "altenate",

// After
"rel":   "alternate",
```

---

### 5. No-header warning — `WriteHTML`

When `gen.Header == ""`, `WriteHTML` prints to stderr:

```
warning: aggregate page has no <h1>; set a 'header' value in the generator YAML
```

This is printed once per `WriteHTML` call, not per item.

---

## CSS impact summary

| Change | CSS impact |
|--------|-----------|
| Skip link | New `.skip-link` / `.skip-link:focus` rules needed in default CSS |
| `<address>` → `<footer>` | `article address` selectors must become `article footer` |
| Date below heading | `article h2` selectors may need adjustment if they styled inline date text |
| `<time>` element | New `article footer time` or `article p time` selectors for date styling |

These impacts must be noted in the aggregation list documentation page on the website.

| File | Change |
|------|--------|
| `html.go` | All changes: `Generator.Lang`, skip link, `WriteItem` structure, typo fix, warning |

## HTML examples

### Post page `<body>` opening (after):

```html
<body>
  <a href="#main-content" class="skip-link">Skip to main content</a>
  <nav aria-label="Site navigation">
    …
  </nav>
  <main id="main-content">
    <article data-published="2026-04-15" data-link="…">
      <h1>Post Title</h1>
      <p>By Author, <time datetime="2026-04-15">2026-04-15</time></p>
      …
    </article>
  </main>
  …
</body>
```

Note: the "By Author, DATE" line on post pages is part of the rendered Markdown body, not
generated by the HTML template. The `<time>` element in post body text is not in scope for
this change — that comes from the author's Markdown.

### Aggregate collection page — single item card (after):

```html
<article data-published="2020-04-11" data-link="https://example.com/post" data-pagefind-filter="…">
  <h2>Post Title</h2>
  <p><time datetime="2020-04-11">2020-04-11</time></p>
  <p>Short description of the post.</p>
  <footer>
    <a href="https://example.com/post">https://example.com/post</a>
  </footer>
</article>
```
