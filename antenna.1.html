NAME

antenna

SYNOPSIS

antenna OPTIONS ACTION [PARAMETERS]

DESCRIPTION

antenna is a tool of the Skimmer Project for working with feeds and rendering link blogs. It is focus on bring Textcasting to the command line and static website deployments. antenna transforms the functionality of the individual Skimmer project command line utilities into a single command that performs the various actions needed to read and write feeds. This includes generating a static link blog that can be used as a basis for distributed social web activity based on the RSS 2.0 feed format.

ACTION

Actions form the primary way you use Antenna and manage a link blog through its life cycle. Below is a list of supported ACTION and their purpose. ACTION can be split between two general purposes . The following commands are related to curating your Antenna’s collections of feeds and feed items.

init
Initialize an Antenna instances by create an “antenna.yaml” (YAML) configuration file. It holds the list of feed lists managed by your Antenna instance. It also include configuration for where to write the resulting link blog and for how to process it.
add COLLECTION
Add a feed collection to your Antenna instance. If a collection doesn’t have a YAML configuration it will be created. The collection’s YAML configuration is used to format the the collection’s HTML page in the “generate” action.
add COLLECTION FEED_URL
Add a feed to a collection managed by Antenna.
add COLLECTION FEED_URL GUID [FILEPATH]
Add a CommonMark document with front matter to the feed in the collection managed by Antenna. By default it reads from standard input if FILEPATH isn’t provided.
edit COLLECTION
Edit a feed collection to your Antenna instance.
edit COLLECTION FEED_URL
Edit a feed to a collection managed by Antenna.
edit COLLECTION FEED_URL GUID
Edit a single item in a feed in a the collection managed by Antenna. The antenna will use the default terminal based editor on your system to edit the item associated with the GUID and FEED_URL in the collection.
remove COLLECTION
Remove the collection from your Antenna instance
remove COLLECTION FEED_URL
Remove FEED URL from the collection managed by your Antenna instance.
remove COLLECTION FEED_URL GUID
Remove an item associated with the GUID and FEED URL from the collection managed by your Antenna instance.
list
List the collections managed in your Antenna instance
list COLLECTION
List the feeds in the collection managed by your Antenna instance
list COLLECTION FEED_URL
List the collection’s feed url details
manage COLLECTION
This provides a localhost web service for managing collections, feeds and items via a WEB UI in your browser.

The following commands are related to producing a link blog static website.

harvest || harvest COLLECTION || harvests COLLECTION FEED_URL
The harvest action retrieves all collections’ feeds and stores them an the appropriate SQLite3 “.skim” database file. If you specify a collection then the retrieval process is restricted to the feeds defined in that collection. If you specify collection and feed url, then only those items are retieved.
generate || generate COLLECTION
This process the collections rendering HTML pages for each collection. If the collection name is provided then only that HTML page will be generated.
preview
Let’s your preview the rendered Antenna as a localhost website

CONFIGURATION

antenna uses a YAML configuration file. Below the the primary attributes you can set in the YAML file.

port
(optional, default: 8000) The localhost port for the “view” action
htdocs
(optional, default: htdocs) The directory that rendered CommonMark, assets and HTML writen to. This is the directory that will be served out using the “view” action.
page
(optional) This holds the default configuration to use rendering the Antenna collections.
static
(optional) This is the directory that contains CommonMark documents that will be converted into HTML when generating your static website. Files ending in “.md” will become “.html” files written to the htdocs directory.
feeds
(optional) This ia list of collections that will be rendered as their own feed upon website generation.
collections
(required) This holds a list of collections managed by your Antenna instance

The collections attribute holds a collection objects. Each collection object has the following attributes.

name
(required) name of the collection
url_file
(required)
page
(optional) The filepath to the YAML configuration used to render the colllection.
